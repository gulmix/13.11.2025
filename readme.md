# URL Checker

## Описание

Сервис принимает список URL, проверяет их доступность и возвращает статус для каждого. Пользователь может получить уникальный ID для набора ссылок и позже запросить отчёт в PDF с результатами.

## Структура проекта

- `cmd/api/main.go` — точка входа.
- `internal/models` — модели данных.
- `internal/repository` — работа с in-memory хранилищем.
- `internal/service` — бизнес-логика.
- `internal/transport/http/handlers` — обработчики HTTP-запросов.
- `internal/transport/http/router.go` — маршрутизация.

## Стандартные практики

- Сервер запускается с graceful shutdown для безопасного завершения обработки запросов.
- Используется модульная архитектура: модели, сервисы, репозитории, транспорт.
- Для генерации PDF используется библиотека `gofpdf`.

## Паттерны

- Repository pattern — слой доступа к данным.
- Service pattern — слой бизнес-логики.
- Dependency injection — внедрение зависимостей через конструкторы.

## Особенности и ограничения

- Для полной реализации перезагрузки/остановки in-memory хранилища не подходят, поэтому нужно было бы использовать базы данных по типу Postgres, Redis и т. д.

## Запуск

1. Установите зависимости:
- go get -u github.com/gin-gonic/gin
- go get -u github.com/jung-kurt/gofpdf
2. Запустите сервер:
- go run cmd/api/main.go
